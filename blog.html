Discuss the scope of var, let, and const +

    var: 'var' declarations are globally scoped or function/locally scoped. The scope is global when it is declared outside of a function. It is function scoped when it is declared within a function. 'var' variables can be re-declared and updated. If it is use to a variable before declaration, it will give a 'Undefined' message.

    let: 'let' is block-scoped (A block is a chunk of code bounded by {}). 'let' variables can be updated but not re-declared. If it is use to a variable before declaration, it will give a 'Reference Error' message.

    const: 'const' is also block-scoped. 'const' variables cannot be updated or re-declared. If it is use to a variable before declaration, it will give a 'Reference Error' message.

The use cases of null and undefined +

    Null: It is the intentional absence of the value. It is one of the primitive values of JavaScript.

    Undefined: It means the value does not exist in the compiler. It is the global object.

What means by REST API? +

    REST API: An API, or application programming interface, is a set of rules that define how applications or devices can connect to and communicate with each other. A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. For this reason, REST APIs are sometimes referred to RESTful APIs.
